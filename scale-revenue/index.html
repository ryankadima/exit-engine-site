AGENT 1

Revenue Forecast

Most founders forecast from a CRM report and a gut check. This agent replaces that with a probability model that synthesizes three data layers into a weekly forecast you can actually make decisions on.

Goal: Produce a weekly probability-weighted revenue forecast in three scenarios — conservative, likely, optimistic — with a narrative brief explaining what moved and why.

Instructions: Technical Implementation PDF

Agent 1 is the synthesis layer. It doesn't collect data — it reasons over data collected by three agents running underneath it. Agent 2 tracks deal velocity, Agent 3 classifies pipeline health, and Agent 4 maps marketing-sourced pipeline. Agent 1 takes all three feeds, applies a probability model to every open deal, and produces a forecast in three scenarios every Monday morning.

How the Probability Model Works

Claude scores each deal by starting with a stage baseline close rate, then applying signal modifiers from the other three agents:

62% Healthy pipeline + multi-threaded + marketing-sourced deal
78% In legal review + champion actively engaged within 7 days
38% Proposal stage + on-track velocity + single data source
11% Single-threaded + stalled pipeline + outbound-sourced

↳ Agents 2, 3, and 4 write classified outputs to a shared pipeline staging layer

↳ Agent 1 pulls all open deals enriched with sub-agent signal tags

↳ Claude scores each deal with a probability percentage based on stage baseline + signal modifiers

↳ Three forecast scenarios calculated by summing probability-weighted deal values

🤖 Claude Prompt — Deal Scoring System:
You are a revenue forecast model. Score each deal using the exact rules below. Apply rules mechanically — do not interpret or adjust based on your own judgment. Return only valid JSON.

STAGE BASELINE CLOSE RATES:
[Prospecting: 8%, Qualification: 15%, Discovery: 28%, Proposal: 38%, Negotiation: 61%, Contract Review: 79%]

SIGNAL MODIFIERS (additive to baseline):
Positive: stakeholder_count >= 3: +15% | last_contact_days <= 7: +12% | pipeline_classification = healthy: +10% | pipeline_source = marketing: +10% | velocity_tag = on_track: +8%

Negative: stakeholder_count = 1: -18% | pipeline_classification = stalled: -20% | pipeline_classification = ghost: -35% | close_date_slips >= 2: -15% | last_contact_days > 14: -12% | pipeline_source = outbound: -8%

HARD OVERRIDES (applied after modifier math):
stage = Contract Review AND last_contact_days <= 7: floor 70%
pipeline_classification = ghost: ceiling 15%
days_to_close < -90: ceiling 5%, set flag: past_due

Return JSON: { "deal_id", "probability": 0-100, "tier": "hot|warm|cold", "reasoning": "One sentence citing the 2 most influential signals.", "override_applied": true/false, "flag": "past_due|requires_review|null" }

AGENT 2

Deal Velocity

Most founders find out a deal is dying when it shows up as a miss at end of quarter. By then, there is nothing to do. This agent catches slipping deals in real time — weeks before they become a forecast miss.

For custom implementation book a call.

Goal: Build a velocity fingerprint from your closed-won history and run every open deal against it daily, flagging risk before deals become forecast misses.

Instructions: Technical Implementation PDF

It works in three stages. First, it builds a velocity fingerprint from your closed-won history — average days per stage, stakeholder count by stage, email cadence in wins vs. losses. Then it runs every open deal against that fingerprint daily. Finally, it classifies each deal by risk level, auto-tags your CRM, and pushes alerts with AI-generated next steps to Slack. Your reps get coached on what to do before they even know there is a problem.

↳ Export closed-won and closed-lost deals to build the velocity fingerprint

↳ Store the fingerprint as a baseline model your automation layer queries daily

↳ Claude compares each open deal against the fingerprint and returns a structured risk assessment

↳ At-risk and critical deals get Slack alerts with specific, actionable next steps

What the Analysis Catches

↳ Deal stuck in stage: Deal A has been in Evaluation for 23 days. Fingerprint says closed-won deals average 14 days. Claude flags "stuck" with severity: high.

↳ Single-threaded deal: Deal B is in Proposal with only 1 contact. Fingerprint says closed-won deals average 3.5 stakeholders by Proposal. This pattern matches 78% of closed-lost deals.

↳ Champion disengaging: Deal C's champion response time went from 2 hours to 5 days. The 60x increase in response time is the strongest predictor of deal death in most B2B pipelines.

🤖 Claude Prompt — Deal Velocity Analysis System:
You are a deal velocity analyst for a B2B sales team. You will receive two inputs:

1. VELOCITY FINGERPRINT — baseline metrics derived from closed-won deals (avg days per stage, stakeholder count by stage, champion response cadence)
2. OPEN DEAL DATA — current metrics for a single open deal

Compare the open deal against the fingerprint. Identify any metric where the deal deviates significantly from the healthy baseline.

Rules:
- Flag "stuck" if days_in_stage exceeds the fingerprint average by more than 50%
- Flag "single-threaded" if stakeholder_count is below the fingerprint minimum for that stage
- Flag "champion disengaging" if champion avg response time has increased by more than 2x from their own earlier baseline on this deal
- Flag "gone dark" if zero email activity for 7+ days in an active stage
- A deal with zero flags is "healthy" | 1 flag is "at_risk" | 2+ flags is "critical"

Return valid JSON only: { "deal_id", "deal_name", "risk_level": "healthy | at_risk | critical", "risk_signals": [{ "signal", "expected", "actual", "severity" }], "velocity_score": 0-100, "summary": "2-3 sentence assessment" }

AGENT 3

Pipeline Visibility

Most founders pull a stale CRM report and call it pipeline. Your CRM says $1.2M. $400K hasn't responded in three weeks. That's not real pipeline. This agent builds a living snapshot that tells you what's real without asking anyone.

For custom implementation book a call.

Goal: Classify every open deal as healthy, stalled, ghost, or hidden — so you see what's real, what's dead, and what pipeline your CRM is missing entirely.

Instructions: Technical Implementation PDF

Your enrichment tool pulls intent signals for every open deal. Your automation layer reconciles CRM, email, and calendar activity. Claude classifies each deal into one of four states with a confidence score. The pipeline delta is calculated — reported vs. real pipeline value — and the classification is written back to your CRM. The result is surgical clarity, delivered daily.

The Four Deal States

Healthy Activity matches stage. Meetings happening. Champion responding. Continue current approach.
Stalled Stage says active, signals say dead. No reply 12+ days, no meeting scheduled. Rep intervention needed.
Ghost Zero engagement in 21+ days across all channels. Near-zero close probability. Escalation or removal.
Hidden Buying signals in email threads but no CRM opportunity. Found pipeline, not lost pipeline.

↳ Clay enriches every open deal with intent signals, technographic data, and firmographic context

↳ n8n reconciles CRM stage data against email, meeting, and calendar activity

↳ Claude classifies each deal: Healthy, Stalled, Ghost, or Hidden — with confidence score and reasoning

↳ The pipeline reality delta is calculated: Reported pipeline minus Ghost and Stalled = what's actually alive

The pipeline reality check: Your CRM says $1.2M in pipeline. After classification: $400K is Ghost or Stalled. Your real pipeline is $800K. That's a 33% gap your team is currently pretending doesn't exist. This agent surfaces it automatically, every day.
🤖 Claude Prompt — Deal Classification System:
You are a pipeline intelligence analyst. Your job is to classify open deals based on real activity signals — not CRM stage labels.

You must classify each deal into exactly one of four states:

1. HEALTHY — Activity matches the deal's current stage.
Signals: champion replied within 7 days, meeting in last 14 days or next meeting scheduled, activities exist since last stage change.

2. STALLED — CRM stage says active, but engagement signals say dead.
Signals: no champion reply in 12+ days, no meeting in 14+ days and none scheduled, stage moved but no supporting activity.

3. GHOST — Complete disengagement for 21+ days.
Signals: no inbound email in 21+ days, no meeting in 21+ days, no response to last 2+ outreach attempts.

4. HIDDEN — Buying signals exist but no CRM opportunity.
Signals: email threads with buying language, meetings with prospect contacts, but deal_id is null or stage is blank.

Rules:
- Base classification ONLY on the activity data provided.
- If signals conflict, weight recency: a reply yesterday overrides 21 days of silence before that.
- Ghost overrides Stalled (21+ days of zero engagement = Ghost).
- Assign a confidence score from 0.0 to 1.0. Above 0.8 = high certainty. Below 0.5 = flag for human review.

Return valid JSON only: { "deal_id", "deal_name", "deal_value", "classification": "HEALTHY | STALLED | GHOST | HIDDEN", "confidence": 0.0-1.0, "reasoning": "2-3 sentences", "key_signals": [], "recommended_action", "days_until_risk_escalation": number_or_null }

Marketing-Sourced Pipeline

Goal: Stitch together every channel touch — from first LinkedIn click to closed deal — into a single contact journey, then score which channels and content actually predict pipeline.

Instructions: Technical Implementation PDF

The agent pulls contact journeys from your CRM, LinkedIn engagement from Fibbler, and webinar attendance from Riverside. n8n stitches them into a single chronological journey per deal. Claude scores which channels and content pieces show up most in pipeline-generating journeys and writes a weekly brief: how much pipeline was created, which channels sourced it, which content appeared in winning journeys, and where you're wasting spend.

↳ CRM provides contact timelines, deal source, and close data for the last 7 days

↳ Fibbler maps LinkedIn ad impressions and engagements to named accounts — filling the dark funnel

↳ n8n stitches all signals into a chronological journey object per deal

↳ Claude builds a weighted channel scoring model and writes a Slack-ready weekly brief

Why this matters: LinkedIn is heavily underreported in most CRMs. A contact might see your ad 12 times, never click, then respond to an outbound email 3 weeks later. Without Fibbler, that deal gets attributed to outbound. With this agent, you see the full warm-up sequence and weight the channel correctly.
